Class: ColumnSchemaFactory

ColumnSchemaFactory

new ColumnSchemaFactory(columns)

Build, organize, and sort a column schema list from a list of columns.

FilterTree requires a column schema. As a fallback when you don't have a column schema of your own, the string array returned by behavior.getFields() would work as is. This factory object will do a little better than that, taking Hypergrid's column array and creating a more textured column schema, including column aliases and types.

CAVEAT: Set up the schema completely before instantiating your filter state. Filter-tree uses the schema (in part) to generate column selection drop-downs as part of its "query builder" UI. Note that the UI is not automatically updated if you change the schema later.

Parameters:
Name Type Description
columns Array.<Column>
Properties:
Name Type Description
schema Array.<menuItem>

This is the output produced by the factory.

Methods

organize(columnGroupsRegex)

Organize schema into submenus.

Parameters:
Name Type Attributes Default Description
columnGroupsRegex RegExp

Schema names or aliases that match this are put into a submenu.
options.key string <optional>
 'name'

Must be either 'name' or 'alias'.

sort(prefixopt)

Sort the schema.

Walk the menu structure, sorting each submenu until finally the top-level menu is sorted.

Parameters:
Name Type Attributes Default Description
prefix string <optional>
 ''

A submenu sort prefix:

  • Omit to give no special treatment to submenus.
  • Give '\u0000' to place all the submenus at the top of each enclosing submenu.
  • Give '\uffff' to place all the submenus at the bottom of each enclosing submenu.
